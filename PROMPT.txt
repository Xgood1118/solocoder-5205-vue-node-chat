客服组需要一个内部会话管理工具，跟踪每个客户咨询的处理状态。前端 Vue 3，后端 Node.js 配 Express，端口走环境变量。数据全在内存就行，进程重启数据丢没事。

后端接口：POST /api/sessions 新建会话、GET /api/sessions 列表（带 status/assignee/category 筛选）、GET/PATCH /api/sessions/:id 详情和改状态、POST /api/sessions/:id/messages 发消息、POST /api/sessions/:id/assign 指派、POST /api/sessions/:id/finish 完结。状态变更走单独接口 /api/sessions/:id/transition，服务端校验状态机合法。超时检测用 setInterval 每分钟扫一遍。

每个会话（也就是一次客户咨询）有客户姓名、联系方式（手机或邮箱）、问题分类（产品咨询/技术支持/投诉/退换货/其他）、创建时间、当前状态、负责人、消息历史这几个字段。状态有 5 种：待分配、已分配、处理中、待客户回复、已完结。

状态流转按这个走：新建会话后状态是"待分配"；管理员指派给某个客服后变"已分配"；客服第一次发消息后变"处理中"；客服发消息后等客户回复过程中变"待客户回复"；客户回复后变回"处理中"；任意时刻客服都可以点"完结"变"已完结"；"已完结"可以重新打开回到"处理中"；"已分配"如果没被处理可以撤销回"待分配"。这些跳转要服务端的合法转换矩阵校验，不合法的返回 409 Conflict。

消息记录比较简单：发送方（客户/客服）、内容、时间、是否内部备注（只有客服能看到，备注用于客服之间沟通）。客户消息模拟就行，UI 上有个"模拟客户回复"按钮，点了之后随机生成一条客户消息加进历史。

有个超时机制：会话处于"已分配"超过 2 小时还没人开始处理（没发消息），自动变回"待分配"；会话处于"待客户回复"超过 48 小时没动静，自动变"已完结"。这两个超时检测用定时器或者轮询实现都行。超时要写一条系统消息进消息历史，标记"已超时自动转单"。

会话列表页要能按状态、负责人、分类筛选。详情页时间轴展示所有消息和状态变更，颜色块区分系统消息/客户消息/客服消息/内部备注。导出功能：把所有已完结的会话按时间范围导成 JSON 或者 CSV，CSV 字段要含全部状态变更点。

还有个内部备注的功能，客服可以在会话里加备注只给自己看，前端展示上要明显标"内部备注"且用不同背景色。客户那边看不到备注内容。整站跑起来要能新建会话、分配、对话、改状态、超时自动转、改状态有时间线展示。npm 启动能起来就行。

补充需求：客服能看到自己正在处理的会话数（按状态分），超时提醒能在右上角弹通知（前端用 EventSource 或 WebSocket 拉后端 /api/notifications 长连接）。会话详情页要显示完整 SLA：响应时间（首次响应距离创建多久）、处理总时长。

再加：会话支持"标签"功能，客服能给会话打多个标签（如"投诉"+"加急"+"老客户"），按标签筛选用。客户消息支持富文本（图片、链接），前端显示图片和链接预览。客服能"转移"会话给其他客服（带转移原因）。

会话支持"优先级"标记——客服能给会话打高/中/低优先级，高优先级会话在列表里标红色并排在前面。会话能"批量处理"——多选后批量完结、批量改分类、批量转给其他客服。

会话要支持"满意度调查"——已完结的会话弹一个 1-5 星评价邀请，评价完才能算真正完结。客服工作台要展示"今日处理量"和"平均处理时长"统计。

会话要支持"语音消息"模拟（前端用 Web Speech API 转文字存消息内容，标识 isVoice 标记）。客服工作台首页要展示"今日新增会话数"、"处理中会话数"、"待客户回复会话数"统计。
